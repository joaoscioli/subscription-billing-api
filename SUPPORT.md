# Support

This repository is maintained as a professional portfolio backend project.
Questions, bug reports, and improvement suggestions are welcome when they help
make the API easier to understand, run, test, or review.

## Best Ways To Ask For Help

- Open an issue for reproducible bugs, documentation gaps, or architecture
  questions.
- Include the affected endpoint, command, test, or document when possible.
- Share expected behavior, actual behavior, and the smallest reproduction you
  can provide.

## Good Issue Examples

- A local setup step is missing or unclear.
- A REST endpoint has inconsistent validation behavior.
- A test does not explain the business rule it protects.
- A documentation page does not match the current implementation.

## Out Of Scope

- Production support for a real billing system.
- Requests involving private credentials, customer data, or paid services.
- Large feature requests without a clear learning or portfolio goal.
