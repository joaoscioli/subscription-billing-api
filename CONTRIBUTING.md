# Contributing

This repository is maintained as a portfolio-grade backend project.

## Working Style

- Prefer small pull requests with one clear objective.
- Keep changes vertical when possible: migration, domain, API, tests, and docs.
- Update documentation when behavior, setup, or architecture changes.
- Keep commit messages clear and scoped.

## Before Opening A Pull Request

- Run the test suite.
- Review API behavior and error responses.
- Check whether an ADR or documentation update is needed.
- Use the pull request template as a review checklist.

## Review Focus

Reviewers should look for correctness, maintainability, test coverage,
security impact, and clear API contracts.
